# 📌프로젝트 알고리즘 작동 방식
- 프로젝트의 알고리즘 작동 방식은 사용자 입력 처리에 초점을 맞추고 있습니다.

데이터베이스 테이블은 두 개입니다. 첫 번째 테이블는 레시피 정보를 저장합니다. 

이 테이블은 레시피의 고유 번호(id), 이름(title), 이미지 URL(img_url), 조리 시간(timecost), 난이도(difficulty), 조리 단계(cookstep), 작성자 ID(author_id), 생성 및 업데이트 날짜(created_at, updated_at) 그리고 메인 및 서브 재료(main_ingredients, sub_ingredients), 정규화한 메인 및 서브 재료(n_main_ingredients, n_sub_ingredients)등을 포함합니다. 

두 번째 테이블은 식재료 번호에 따라 레시피 id를 저장합니다. 

식재료 사전을 사용해 식재료들은 모두 고유의 ‘식품명 id’를 가지고 있다. 이 테이블은 식품명 id(ingre_id), 해당 식재료가 메인재료로 사용된레시피 id(main_recipe_id), 해당 식재료가 서브재료로 사용된 레시피 id(sub_recipe_id)입니다.

사용자 입력 데이터는 이 데이터는 레시피 식별, 분류 및 추천에 활용됩니다.

첫 번째 테이블에 사용한 초기 데이터는 '만개의 레시피' 웹사이트에서 크롤링한 것입니다. 

사용자 입력 데이터는 레시피명, 조리 시간, 난이도, 조리 과정, 이미지 및 재료 정보를 포함합니다. 

그리고 조리과정 분류 알고리즘을 통해 레시피의 조리분류를 자동으로 결정하여 테이블의 classfication열에 데이터를 저장한다. 또한 사용자가 입력한 재료를 정규화하는 알고리즘을 통해 정규화한 재료를 n_main_ingredients와 n_sub_ingredients열에 저장합니다.

추천 알고리즘은 두 단계로 구성됩니다. 

첫 번째 단계에서는 유사한 재료를 사용하는 레시피를 추천하며, 메인 재료와 서브 재료의 유사도에 따라 추천 순위를 결정한다. 두 번째 단계에서는 첫 번째 단계의 단점을 보완하여 조리 분류(classification)를 고려한 레시피 추천을 진행합니다.

'조리 분류 유사 레시피 추천 알고리즘'은 기존 레시피 데이터에서 동사와 조리 분류 간의 관계를 분석하고, 각 조리 분류별로 동사의 출현 빈도를 계산하여 조건부 확률을 도출합니다. 

이 확률은 특정 동사가 주어졌을 때, 그 동사가 특정 조리 과정에 속할 확률을 나타냅니다. 

TF-IDF 기법을 사용하여 빈도가 낮은 동사의 문제를 해결하고, 데이터의 표준편차 역수를 이용해 가중치를 계산한다. 이를 통해 조건부 확률과 TF-IDF를 통합하는 가중평균을 구하며, 이를 기반으로 각 조리 분류별 상위 100개의 동사 리스트를 생성합니다.

식재료 사전은 사용자 입력의 정규화를 위해 구축되며, 식재료의 다양한 표현을 표준화합니다. 

이 사전은 식품명, 품종, 종류, 가공형태 등 다양한 열로 구성되어 있으며, 사용자 입력의 다양성에 대응하기 위해 사용된다. 이 사전을 사용하여 메인재료와 서브재료를 정규화하며, 정규화된 결과는 데이터베이스의 정규화 열에 저장됩니다.

마지막으로, 재료 기반 유사 레시피 추천을 위해 메인 재료와 조리 과정을 겹치는 레시피를 추출하고, 이 중에서 서브 재료가 가장 많이 겹치는 순서로 레시피를 추천합니다.

### 추천 알고리즘

- 첫 번째 단계에서는 유사한 재료를 사용하는 레시피를 추천하며, 메인 재료와 서브 재료의 유사도에 따라 추천 순위를 결정합니다.

# 개선 내용
1. 데이터 처리 및 관리 방식

데이터 정규화를 통해 더 정확한 레시피 추천을 가능하게 하고, 신규 사용자 데이터와 기존 데이터를 통합 관리하여 더 넓은 범위의 데이터를 활용합니다.

메인 재료와 서브 재료를 정확히 분류 및 정규화하는 과정과 고급 데이터 분석 기법을 적용하여 조리과정을 분류하는 과정을 거쳤습니다. 

이후 1단계에서는 유사한 재료를 사용하는 레시피 중 메인 재료와 조리 과정이 겹치는 것을 추출하고 서브 재료가 많이 겹치는 순으로 추천하며, 2단계에서는 이를 보완하여 재료 기반으로 선별된 후 조리 과정을 기준으로 더 정확한 추천을 제공하는 이중 추천 시스템을 도입하여 추천의 다양성과 정확성을 강화했습니다. 

3. 웹페이지 구현
